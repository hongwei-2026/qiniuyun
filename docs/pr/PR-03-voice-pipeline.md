# PR-03：语音管道稳定性与漫画/空间指令修复

**合并方向：** `integrate` ← `pr/03-voice-pipeline`  
**对应提交：** `720a329`（基于 PR-02）

## 功能描述

修复评测与自测中发现的语音交互问题：

- **转写与采音解耦**：上一句转写进行中仍可说下一句（队列上传）
- **Q 键**：仅开关语音识别，不再承担「提交指令」
- **漫画项目切换**：支持「选择漫画二」等口语，修复误匹配默认项目
- **空间解析**：支持「在左上角画…」类位置描述
- **频控**：不再自动关麦，避免「说几句后无反应」
- **讯飞 OST**：预录回溯、超时对齐、转写间隔优化
- **碎片过滤**：单独「正方形」等不触发；`transcriptFilter` 去重
- **后端**：转写轮询加速；`transcribe` 异常不 500

## 实现思路

- `xfyunOst.ts`：`transcribeChain` 队列 + 独立 `chunks` 缓冲
- `useVoicePipeline.ts`：watchdog 自愈、`ensureListeningActive` 强化
- `voiceCommands.ts` / `comicEngine.ts`：`选择` 动词与序号解析
- `spatialEngine.ts`：`在左上角` 锚点正则

## 测试方式

1. 连续说 3 条指令，确认无需等待转写完成即可说下一句
2. 漫画模式：「选择漫画二」应切换到对应项目
3. 「在左上角画红色圆形」识别区含位置且落点正确
4. 按 Q 仅开关麦，说完一句自动执行
